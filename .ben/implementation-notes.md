# Implementation Notes

How each requirement was fulfilled in code. Cross-referenced to requirement IDs in `requirements.md`.
Use this file to understand the exact approach taken so changes can be reproduced or extended without trial and error.

---

## General Conventions (apply to all requirements)

- All AI-generated CSS overrides go at the **bottom** of `index.css` inside `/* AI: below section was generated by an AI */ … /* AI: end */`. Never scatter overrides through the file.
- **Never remove HTML IDs** even when hiding elements — JS references IDs by string and will throw if they are missing.
- **Hidden nav buttons** must stay in the DOM — remove them visually with `display:none` but keep the HTML so JS array indexing (`tablinks[2]`, `tablinks[3]`) does not break.
- **Deploy rules for this device** (LittleFS has been populated — cfg.json, presets.json etc. are on-device): WLED's HTTP server checks LittleFS first and serves files from there, ignoring the firmware-embedded copies. This means **web UI changes always require cmd 3** on this device — cmd 1 + 2 alone will not take effect. Pure C++ changes still only need cmd 2. A fresh device with an empty LittleFS would fall back to firmware-embedded files and not need cmd 3, but that does not apply here.
- ESP32-S2 Mini always needs a **manual RST button press** after flash — the USB reset fails and pio prints "FAILED", which is normal.

## REQ 1.3.4 / 1.3.5 / 1.3.6 — Device name, AP always-on, UI defaults

**Files:** `wled00/data/cfg.json`, `wled00/data/index.js`
**Approach:**
- Device name: set `id.name` to `""` in cfg.json.
- AP always-on: set `ap.behav = 2` in cfg.json (value 2 = `AP_BEHAVIOR_ALWAYS`). Without this the AP only activates when no router connection is available.
- UI defaults (colour wheel, sliders, selectors, hex, dark mode): these are **browser localStorage** values, not server-side config. They are set via `cfg.comp.colors` defaults in `index.js`. They only take effect on a new or cleared browser session — a returning browser with cached localStorage will keep its own values regardless of what cfg.json says.

**cfg.json field reference:**
```
id.name           — device name (blank = empty string)
ap.behav          — 0=no AP, 1=AP if no router, 2=AP always on
hw.led.ins[].pin  — LED data GPIO
hw.led.ins[].len  — pixel count
hw.led.ins[].type — LED type (22 = WS2812_RGB)
hw.led.maxpwr     — ABL limit in mA (0 = disabled)
hw.btn.ins[]      — button config (type 2 = pushbutton)
light.tr.dur      — transition duration in units of 100ms (7 = 700ms)
def.ps            — boot preset ID
```
**Gotcha:** The UI settings (colour wheel, sliders etc.) are localStorage-only. If a user has previously visited the WLED UI on their phone, their cached preferences override the firmware defaults. The only way to reset them is to clear the browser's site data for the device IP.
**Deploy:** Requires cmd 3 (uploadfs) for cfg.json changes. index.js changes require cmd 1 + cmd 2.

## REQ 2.3 — On AP disconnect revert to preset 101

**Files:** `wled00/wled.h`, `wled00/network.cpp`, `wled00/wled.cpp`
**Approach:** Added a `volatile bool revertToDefaultPreset` flag declared in `wled.h`. The flag is set to `true` in `network.cpp` inside the `ARDUINO_EVENT_WIFI_AP_STADISCONNECTED` WiFi event case. The flag is checked and consumed in `wled.cpp` `loop()` after `handleConnection()` — if true, calls `applyPreset(101)` and resets the flag.
**Gotcha:** WiFi events run in the ESP32 system task, not the Arduino main task. You cannot call `applyPreset()` directly from the event handler — it will crash or silently fail. The volatile flag bridges the two tasks safely.
**Deploy:** Requires cmd 2 (firmware upload, C++ only change).

## REQ 3.1 / 3.1.1 / 3.1.2 — Segments, Presets, Info buttons on Config page

**Files:** `wled00/data/settings.htm`, `wled00/data/index.js`, `wled00/data/index.css`
**Approach:** A `.debug-pills` strip added immediately below the Back button in `settings.htm`. Each pill is a styled `<a>` tag. Two deep-link mechanisms used:
- URL hash for tabs with existing hash support: `window.location.origin + '/#Effects'`
- Query param for tabs that needed a new route: `window.location.origin + '/?tab=segments'`, `/?tab=presets`, `/?tab=info`
- External WLED settings routes used directly for Time and Sync: `/settings/sync`, `/settings/time`

The `?tab=` parser added to `onLoad()` in `index.js`:
```js
var tabParam = new URLSearchParams(window.location.search).get('tab');
if      (tabParam === 'presets')  openTab(3, true);
else if (tabParam === 'segments') openTab(2, true);
else if (tabParam === 'info')     toggleInfo();
```
`force = true` argument passed to `openTab()` to bypass the "already on this tab" guard.

Two new cases added to `handleLocationHash()`: `#Timer` → `openTab(0, true)` and `#Info` → `toggleInfo()`. All existing hash cases upgraded to use `force = true`.

**REQ 3.1.1 — Pill style:** `background: #4a4a00; color: #ffd700; border-radius: 20px; font-size: 3.5vmin` — compact, touch-friendly, dark gold.

**REQ 3.1.2 — Default tab:** Changed in two places in `index.js`: `onLoad()` changed from `updateTablinks(0)` to `openTab(1, true)`, and `togglePcMode()` changed to `openTab(1, true)` when entering PC mode. Tab index 1 = Effects.

**Gotcha:** The `force = true` argument is required on `openTab()` — without it the "already on this tab" guard prevents the tab from activating when navigating back from the Config page.
**Deploy:** Requires cmd 1 + cmd 2.

## REQ 3.15 — Git commit ID on Info page

**Files:** `pio-scripts/set_metadata.py`, `wled00/wled_metadata.cpp`, `wled00/wled_metadata.h`, `wled00/json.cpp`, `wled00/data/index.js`
**Approach:** Five-file chain: `set_metadata.py` calls `get_git_commit()` and injects `WLED_GIT_COMMIT` as a compiler define scoped only to `wled_metadata.cpp`. `wled_metadata.cpp` defines `gitCommitString` from that flag. `wled_metadata.h` declares it. `json.cpp` serialises it into `/json/info` as the `commit` field. `index.js` reads `i.commit` in `parseInfo()` and renders it as an info row: `${i.commit ? inforow("Commit", i.commit) : ""}`.
**Gotcha:** If `get_git_commit()` fails silently in `set_metadata.py` (e.g. git not available in the build environment), `WLED_GIT_COMMIT` is undefined and the Info page shows "unknown" with no build error surfaced.
**Deploy:** Requires cmd 2 (C++ change — no web UI source change needed beyond index.js which is a cmd 1 + cmd 2).
