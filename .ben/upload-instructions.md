# Upload Instructions

Always run from inside the project folder:

```
cd "WLED-Main-2026-04-29"
```

---

## What to run and when

### Web UI change (edited anything in wled00/data/)
Run both, in order:

```
npm run build
```

```
& "$env:USERPROFILE\.platformio\penv\Scripts\pio.exe" run -e lolin_s2_mini --target upload
```

---

### C++ / .h / .ini change only (no web UI files touched)
Run this one only:

```
& "$env:USERPROFILE\.platformio\penv\Scripts\pio.exe" run -e lolin_s2_mini --target upload
```

---

### Config or presets change (edited cfg.json, presets.json, or wsec.json)
Run this one only:

```
& "$env:USERPROFILE\.platformio\penv\Scripts\pio.exe" run -e lolin_s2_mini --target uploadfs
```

---

### Everything changed (full redeploy)
Run all three, in order:

```
npm run build
```

```
& "$env:USERPROFILE\.platformio\penv\Scripts\pio.exe" run -e lolin_s2_mini --target upload
```

```
& "$env:USERPROFILE\.platformio\penv\Scripts\pio.exe" run -e lolin_s2_mini --target uploadfs
```

---

## Other device commands

### Wipe device (erases everything)

```
~\.platformio\penv\Scripts\platformio.exe run --target erase -e lolin_s2_mini
```

---

## Notes
- ESP32-S2 Mini always needs a manual RST button press after flash — pio printing "FAILED" is normal
- uploadfs and firmware upload are independent — either can be run without the other
- Never edit wled00/html_*.h or wled00/js_*.h — they are auto-generated by npm run build
