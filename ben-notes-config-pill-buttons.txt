Approach: Segments & Pill Buttons on the Config Page
1. The Pill Buttons in settings.htm
A .debug-pills strip was added immediately below the "Back" button on the settings landing page. Each pill is a styled <a> tag that navigates using two different deep-link mechanisms:

Mechanism A — URL hash (for named tabs that already had hash support):


window.location.origin + '/#Effects'
Mechanism B — Query parameter ?tab= (for tabs that needed a new route):


window.location.origin + '/?tab=segments'
window.location.origin + '/?tab=presets'
window.location.origin + '/?tab=info'
The pills navigate to external WLED settings routes (/settings/sync, /settings/time) directly for those two.

CSS: Dark gold pill style (background: #4a4a00, color: #ffd700, border-radius: 20px) — compact, touch-friendly, 3.5vmin font size.

2. Teaching index.js to Respond to ?tab=
In onLoad(), after the default tab is set, the query string is parsed:


var urlParams = new URLSearchParams(window.location.search);
var tabParam = urlParams.get('tab');
if      (tabParam === 'presets')  openTab(3, true);
else if (tabParam === 'segments') openTab(2, true);
else if (tabParam === 'info')     toggleInfo();
The force = true argument on openTab() bypasses the "already on this tab" guard so it always activates the correct tab on load.

3. Extending the Existing Hash Router
handleLocationHash() already handled #Effects, #Colors, etc. Two new cases were added:


case "#Timer": openTab(0, true); toggleNl(); break;
case "#Info": toggleInfo(); break;
And all existing cases were upgraded to use force = true.

4. Default Tab Change
The UI was reconfigured to open on the Effects tab by default (tab index 1) instead of Colors (tab index 0). This appears in two places:

onLoad(): openTab(1, true) instead of updateTablinks(0)
togglePcMode(): openTab(1, true) when entering PC mode
5. UI Declutter (context for why the pills were needed)
The top button bar and bottom tab bar were hidden (style="display:none") along with the brightness slider, color picker controls, and other chrome. Because the standard navigation was removed, the pill strip on the settings page became the primary way to deep-link into specific tabs (Segments, Presets, etc.) from the config page.

Porting Checklist
To replicate in another build, you need all five parts:

File	Change
settings.htm	Add .debug-pills CSS + <div class="debug-pills"> HTML block
index.js	onLoad() query-param parser + handleLocationHash() new cases + default openTab(1, true)
index.htm	(optional) hide UI chrome if desired; move #picker to Effects tab
index.css	Minor #picker margin tweak — cosmetic only
The core portable logic is settings.htm pills + the ?tab= parser in onLoad(). Everything else is presentation.
